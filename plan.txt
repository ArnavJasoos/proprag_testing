## Flow (run.py)

1. Bootstrap, logging (reuse `proprag_poc.logging_setup`), `check_backend()` (fail fast with koboldcpp launch hint: `koboldcpp.exe --model gpt-oss-20b-Q4_K_M.gguf --port 5001 --contextsize 8192`).
2. Stratified subset → manifest → corpus docs (1 doc = 1 chunk, `f"{title}\n{text}"`).
3. Three sequential index phases, each in `IndexPhase`, skipped when artifacts exist:
   a. shared chunk embeddings → BaseRAG cost; b. POC Extractor + GraphBuilder → PropRAG cost; c. GraphRAG extract+summarize+communities+reports → GraphRAG cost. Write `index_usage.json`.
4. Per-question loop (resume-safe via results.jsonl): for each system — `tracker.scope("q::<system>")`, time retrieve → Recall@k via chunk_id→title; time QA (GraphRAG uses `build_qa_context`); EM/F1; append row (incl. raw answer for re-scoring). Per-system try/except so one failure doesn't kill the run (compare.py pattern).
5. `report.build()`; pilot mode additionally prints measured s/call + projected full-run wall time.

Token attribution: BaseRAG/PropRAG index = "index"-scope snapshot deltas around their sequential phases; GraphRAG index = explicit scope inside its own workers; query = per-question scopes on main thread.

4. `python -m benchmark.run` (full 50) — resumable at three levels (SQLite LLM cache / stage checkpoints / results.jsonl). Deliverable: `data/benchmark/<run_id>/report.md` + metrics.json + charts.

## Risks

- **RAM**: 20B Q4 (~11.5GB mmap) + bge-large (~1.3GB); mitigations above.
- **Runtime**: ~1,400–1,700 chat calls on CPU 20B → potentially multi-day; pilot projects it; per-call max-token caps (extraction 1500 / reports 1000 / QA 512); every call cached so interruptions cost nothing. Index wall time itself is a reported finding.
- **Small-model JSON**: multi-tier repair proven in POC; failure counts surfaced in report.
- **EM sensitivity**: short-span prompt + `Answer:` extraction + normalization; raw answers preserved.
